# AOPM

<pre>
    A     O  P       M
    |     |  |       |
    Axok!_OS Package Manager
</pre>

**AOPM (Axok!\_OS Package Manager)** is a simple and straightforward package manager made for Axok!\_OS.

## Features

- Built with Python 3 🐍
- Simple and transparent 🪟
- Modular and expandable 🧩

## ⚠️ Project Status

AOPM is under active development and **not ready for production use**.

Bugs, incomplete features, and breaking changes are expected.
Use it at your own risk.

## How to install AOPM

### Removing old builds

Before compiling AOPM, it is recommended to remove old builds.

You can do this with:

```bash
make clean
```

Or manually:

```bash
rm -rf compile .config install.config
```

### Configuring the AOPM installer

To configure the installer, use the `configure.sh` script:

```bash
./configure.sh
```

With this method, you will need to define some variables:

- `--prefix`: Defines the root path for installation. Other directories are resolved relative to this path, making it the base location for installed files. Some paths (like configuration or variable data) may be handled separately.

- `--config-dir`: Defines the directory where configuration files will be stored. This path is independent from `--prefix`

- `--var-dir`: Defines the directory where variable data will be stored.

- `--with-aopkg`: If true, installs `aopkg` along with AOPM. The `aopkg` tool helps developers create AOPM packages (`.aopkg.tar.xz`)

Alternatively, you can run:

```bash
make configure
```

This runs `configure.sh` with default values:

```bash
./configure.sh --prefix=/usr --with-aopkg=true --config-dir=/etc/aopm.d --var-dir=/var/aopm
```

You can override these values by editing the `CONFIGURE_FLAGS` variable:

```bash
make configure CONFIGURE_FLAGS="--with-aopkg=true"
```

Note: `--with-aopkg` has a default value, but you must still define the other variables.

### Compiling AOPM API

AOPM uses the aopmAPI, responsible for printing messages, checking API headers, and performing basic GPG verification

To compile the aopmAPI:

```bash
python3 compiler.py .config
```

This runs `compiler.py` using the `.config` file generated by the configure step.

Alternatively:

```bash
make compile
```

You can specify a different config file:

```bash
make compile CONFIG_FILE=".config"
```

The compiler also generates shortcuts for AOPM and aopkg.

### Installing AOPM

To install AOPM, run the `install.py` script using the `install.config` file.

This file is generated by `compiler.py`

```bash
python3 install.py install.config
```

Or:

```bash
make install
```

You can specify a different file:

```bash
make install INSTALL_FILE="install.config"
```

---

After installation, you can test AOPM with:

```bash
aopm help me
```

If everything is working correctly, you should see:

```aopm
AOPM Usage:
----------------------------------
aopm [Method/Module] [Parameters]
----------------------------------
Examples:
    aopm help me - Display this message
    aopm help module (module name) - Display help for the specified module
```

If you encounter any issues installing AOPM, please contact the developers.
